#### git show
- 显示各种类型的对象
- 根据一个对象id来显示这个对象的具体信息
- 可以很方便的看某次提交的改动，或者树

#### git show-ref
- 可以显示所有包含的引用
- 所谓引用，就是引用的某次提交id，而不是直接指向的某次提交，就类似一个变量。
- 这样就方便操作一些常用的提交id了
- 其中heads代表的是分支，remotes代表的是远程分支在本地的映射，tags代表的是里程碑
- 所有的分支可以在.git/refs文件夹下看到

#### git pack-refs --all
- 打包标头和标签以便高效的存储库访问
- 打包.git/refs目录下的引用文件，放到文本文件.git/packed-refs中
- 把当前所有文件对象打包到.git/objects/pack/目录下的两个文件中。以.pack结尾的是打包文件，以.idx结尾的是索引文件。

#### git的打包
- git对于已SHA1哈希值作为目录和文件名保存的对象有一个术语叫做松散对象。
- git拉取远程仓库时会打包一下，把松散对象打包一下成包文件，降低磁盘空间，按需传输，打包传输，提高效率
- 打包到一个文件中还可以提高文件读取效率，因为这样只用读一个文件就够了。
- 克隆之后的版本库在日常提交中产生的依旧是松散对象
- git使用zlib库对文件进行压缩和解压，这个库历史悠久，使用广泛，但是速度和效率上取不及现代压缩算法。

### git fsck
- 查看到版本库中包含的没有被任何引用关联的松散对象,包括直接关联和间接关联。
- git的版本库是一个树形的链式结构，各个分支维护着一条完整的链，在版本控制的过程中可能会产生很多的废弃链条，和节点。这个命令就是查找出那些还在版本库中的废弃节点。
- head 指向的就是一个引用或者一次提交，表示当前版本库在那个节点上

#### git prune
- 清理没有被任何引用直接或间接关联到的对象
